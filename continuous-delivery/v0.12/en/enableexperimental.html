How to enable experimental features

SUSE® Rancher Prime Continuous Delivery supports experimental features that are disabled by default and that can be enabled by the user.

Enabling/disabling experimental features is done using extra environment variables that are available when deploying rancher/fleet.

Enabling an experimental feature

At the moment we’re writing this document, SUSE® Rancher Prime Continuous Delivery has OCI storage as an experimental feature.

Enabling when installing SUSE® Rancher Prime Continuous Delivery stand-alone

All you need to do is to pass something like:

--set-string extraEnv[0].name=EXPERIMENTAL_OCI_STORAGE \
--set-string extraEnv[0].value=true \

to your helm install or update command.

Please note you have to use --set-string because otherwise the boolean value won’t work as expected.

Enabling when installing SUSE® Rancher Prime Continuous Delivery with Rancher

You can also activate the experimental features in SUSE® Rancher Prime Continuous Delivery when installing Rancher.

The parameters are the same, but you have to add the fleet. prefix.

--set-string fleet.extraEnv[0].name=EXPERIMENTAL_OCI_STORAGE \
--set-string fleet.extraEnv[0].value=true \

Available experimental features

Right now SUSE® Rancher Prime Continuous Delivery supports the following experimental features:

  • EXPERIMENTAL_OCI_STORAGE

v0.12.4 (latest) OCI Storage
